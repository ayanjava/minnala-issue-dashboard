        ❌ Failing tests
          🏃 How to run / regenerate copy-pasteable from your shell in the 0dte-v2 repo root
          • Full pytest run (~948 test files) 
            python -m pytest tests/ -v --tb=line
          • Test summary JSON (for the Tests tile + breakdown chart) 
            python -m pytest tests/ --json-report --json-report-file=reports/test-summary.json
          • Storage / Alembic tests (faster slice) 
            python -m pytest tests/storage/ -v
          • Vitest frontend tests 
            cd frontend && npm test -- --run
          • Coverage JSON (for the Coverage tile + by-package chart) 
            coverage run --source=app,src -m pytest tests/ && coverage json -o reports/coverage.json
          • SonarQube scan (requires sonar-scanner + .env.sonar) 
            sonar-scanner -Dsonar.projectKey=ayanjava_0dte-v2
          • Module progress JSON (rate-limit-safe REST API) 
            python deploy/labels/build_progress_report.py > reports/progress.json
          • 5-proof: tag every open + closed issue missing one+ proof 
            python deploy/labels/backfill_needs_proof.py --state all
          • Commit + push the regenerated reports (what the dashboard reads) 
            git add reports/ && git commit -m 'reports: refresh' && git push origin stable/dev
